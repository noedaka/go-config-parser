# 3. Реализовать утилиту на Go, которая анализирует конфигурационный файл веб-приложения и выявляет потенциально опасные настройки.

### Входные данные

На вход подается YAML или JSON-файл с конфигурацией приложения, например:

* открытый debug mode
* пароль в открытом виде
* использование `0.0.0.0` без ограничений
* отключенная TLS-проверка
* слишком широкие права доступа
* использование устаревших или небезопасных алгоритмов

### Базовые требования


Утилита должна:

* принять путь к файлу первым позиционным аргументом
* распарсить конфиг
* проверить его по заданному набору правил
* вывести список найденных проблем с уровнями:

  * `LOW`
  * `MEDIUM`
  * `HIGH`
* для каждой проблемы дать краткое объяснение и рекомендацию. Если есть хотя бы одна проблема, выйти с ошибкой

Предусмотрены следующие параметры (флаги):
- `-s`, `--silent` - не выходить с ошибкой при наличии проблем
- `--stdin` - прочитать конфигурацию из стандартного потока ввода вместо файла

Приложение должно быть написано с заделом на расширяемость: в будущем могут добавиться новые правила, рекомендации или поддерживаемые форматы.

#### Набор правил

- логирование в debug режиме
- пароль в открытом виде
- использование `0.0.0.0` без ограничений
- отключенный TLS
- использование устаревших или небезопасных алгоритмов

#### Примеры конфигураций и рекомендаций

##### Слишком подробное логирование

```json
{"version": "0.1", "log":{"output":"stdout", "level": "debug"}}
```

Пример рекомендации:

```
LOW: логирование в debug-режиме. Поменяйте режим на более избирательный (info+).
```

##### Слабый алгоритм хеширования

```yaml
version: 2.2
storage:
  digest-algorithm: MD5
```

Пример рекомендации:

```
HIGH: слишком слабый алгоритм - MD5. Замените его на более безопасный.
```

#### Дополнительные задания

- запускать утилиту в качестве http-сервера, который имеет REST API. С помощью API можно также передавать конфигурацию на проверку и получать в качестве ответа массив проблем.
- запускать утилиту в качестве gRPC-сервера. С помощью API можно также передавать конфигурацию на проверку и получать в качестве ответа массив проблем.
- рекурсивный анализ директории с конфигами
- проверка прав доступа через `os.Stat`
