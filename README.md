# Утилита на Go, которая анализирует конфигурационный файл веб-приложения и выявляет потенциально опасные настройки.

Реализованы все дополнительные задания.

## Входные данные

На вход подается YAML или JSON-файл с конфигурацией приложения, например:

* открытый debug mode
* пароль в открытом виде
* использование `0.0.0.0` без ограничений
* отключенная TLS-проверка
* слишком широкие права доступа
* использование устаревших или небезопасных алгоритмов



## Запуск

## CLI
```bash
go run ./cmd/cli/main.go [флаги] <файл|директория>
```
### Флаги 
Флаги:

-s - не завершаться с ошибкой при наличии проблем
--stdin - прочитать конфигурацию из stdin
-r - рекурсивный анализ директории


## HTTP
```bash
go run ./cmd/http_server/main.go
```
Сервер запускается на порту :8080.

### Эндпоинты 
POST	/api/config/recommendations/file - Загрузка файла multipart/form-data (поле config)
POST	/api/config/recommendations/ - Конфигурация в теле запроса (text/plain)

При наличии проблем: 400, тело -- текстовый список проблем и рекомендаций.
При отсутствии проблем: 200, тело -- сообщение "Рекомендации к конфигурации не требуются".


## gRPC
```bash
go run ./cmd/grpc_server/main.go
```
Сервер запускается на порту :9090.

Метод: ConfigRecsService.GetConfigRecsByString
Запрос: ConfigStringRequest { data: string } - строка конфигурации.
Ответ: RecsResponse { recs: string } - текстовый отчёт.
